# KeyTao Review

用于把“查词/加词”从普通编码建议升级为带证据的审词流程。

## 使用原则

- 加词前优先调用 `keytao_prepare_reviewed_add`，不要只用默认 `keytao_encode`。
- 工具会从汉典、萌典、百度百科、维基百科、汉语辞典等来源抽取拼音证据。
- 自动审核使用分层证据：优先采用汉典、辞典、百科等可核验来源；若暂未抓到专门页面，常见现代汉语词、成熟术语或大众明确知晓的实体，也可用“本喵语言常识 + 常用词/实体信号”作为依据。
- 语言常识不能替代编码验证：目标编码必须出现在 `keytao_encode` 返回的候选链中；多音、含义不明、陌生专名或信号互相矛盾时必须交给管理员。
- 多个真实读音会全部展示，用户可以选择其中一个或多个编码。
- 删除项不能自动通过；“删除原位 + 创建同词新编码”的调码可按调码审核。
- 调码如果把某词排到另一个同编码词前面，必须比较常用度；语料、搜索、词典、百科信号一致时才自动通过。
- 更常用/更正确的判断要保守：证据不足或有反向信号时提交给管理员，不要强行自动通过。
- `CSS` / `CSSSingle`（声笔笔/声笔笔单字）不是普通 Phrase 的读音+形码候选链；不能用通用双拼或 `f+ao` 这类声韵猜测判错。声笔笔条目按短码表事实、同码链优先级、词频和结构对齐审查。
- 常见现代汉语词、成语、熟语、成熟术语即使没抓到权威拼音页，只要读音/含义大众明确且编码在 keytao_encode 候选链中，可按“本喵语言常识 + 常用词信号”建议通过。
- “需管理员审核”不等于“不可提交”：只有编码/结构硬冲突、重复写入或后端校验失败才阻止提交；证据不足、歧义、纯删除可提交，但应明确说明提交后等待管理员。

## 工具

- `keytao_prepare_reviewed_add(word)`：返回读音来源、候选编码占位、推荐位置和自动审核资格。
- `keytao_audit_draft_items(items)`：对草稿条目进行自动审核预判，返回 pass 或 needs_admin。
